📙책 리뷰 : **Ch. 5 형식 맞추기**<br>

## 🔍TL;DR (3줄 요약)

1. {코드 형식은 생각보다 정말 중요하다.}
2. {잘 쓴 기사처럼 코드를 풀어나가라.}
3. {적절한 길이-연관성-공백 활용}

## 🤔뭐가 기억에 남음? 헷갈린 건?

&nbsp;코드는 하나의 신문 기사와 같다. 기사에 형식이 중요하듯 코드에도 형식이 중요하다.<br>
왜냐하면 가독성에 지대한 영향을 미치기 때문이다.  

코드는 끊임없이 진화한다. 오늘 구현한 기능이 내일 바뀌기도 한다.<br>
그래서 나 or 동료가 써놓은 코드를 다시 읽어야 할 상황이 계속해서 생긴다.  

그럴때 정말 잘 쓴 기사처럼 위에서 내용을 요약하고 아래로 갈수록 세세한 사항을 일러준다면?<br>
아마 읽는 입장에서도 큰 부담없이 받아들이고 이해할 것이다.<br>
이처럼 기사 형식에 해당하는 코드 스타일은 굉장히 중요하다.<br>
심지어는 앞으로 해당 프로젝트의 유지보수와 확장성에 영향을 미친다.  

&nbsp;저자가 강조한 여러가지 형식 내용이 있었지만 요약하자면 다음과 같다.<br>
- 적절한 길이
    - line 수, 가로 길이는 길지않게
- 연관성
    - 밀접한 함수, 변수는 모아서
- 공백 활용
    - 개념과 개념은 빈 line으로 구분, 수식 또한 space를 활용(IDE가 없앨 순 있음)  
<br>

이와 같은 형식은 곧 코드를 얼마나 **한 눈에 들어오게끔** 하느냐가 포인트이다.

## 읽은 소감

&nbsp;얼마나 보기 좋게 쓴 코드인지는 곧 얼마나 볼 사람을 고려하면서 썼느냐 인것 같다.  

당연한 소리지만 나도 완벽한 형식의 무결점 코드를 쓰고 있지는 않다.<br>
하지만 점차점차 규칙에 맞춰 쓰려는 노력을 하다보면 어느 순간 <u>깔끔하고, 일관되고, 꼼꼼하게</u> 코드를 잘 쓴다는 이야기를 듣는 날이 오면 좋겠다.
